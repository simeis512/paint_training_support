# CLAUDE.md

このファイルは、Claude Code がこのリポジトリで作業する際のガイドラインです。

## モデル運用方針（トークン節約のための役割分担）

トークン消費を抑えつつ品質を保つため、作業をモデルの役割で分担します。

### オーケストレーション層（Fable モデル）

- **担当**: 計画の立案、タスクの分解、設計判断、サブエージェントへの指示出し、成果物のレビュー・統合といったオーケストレーション全般。
- **理由**: 全体を俯瞰して指揮する役割は会話の文脈を長く保持する必要があるため、ここに中核モデル（Fable）を充てる。
- **やらないこと**: 自分で大量のコードを書き続けない。実装は下記サブエージェントに委託する。

### 実装層（Sonnet / Opus サブエージェント）

- **担当**: 仕様が固まった後の実装作業、定型的なリファクタリング、テストの追加、ボイラープレートの生成など、明確に切り出せる作業。
- **使い分けの目安**:
  - **Sonnet サブエージェント**: 単純〜中程度の実装、繰り返し作業、ファイル横断の機械的な変更。コスト効率を優先するデフォルト。
  - **Opus サブエージェント**: 難易度の高い実装、複雑なロジック、慎重な判断を要する箇所のみ。
- **委託の単位**: 各サブエージェントには、それ単体で完結できるよう必要なファイルパス・前提・受け入れ条件を明示して渡す（サブエージェントは親の会話文脈を引き継がない）。

### 運用ルール

1. まず Fable で計画を立て、作業を独立したサブタスクに分解する。
2. 依存のないサブタスクは並列にサブエージェントへ委託する。
3. 簡単な作業ほど Sonnet を優先し、Opus は必要な場面に限定してトークン消費を抑える。
4. サブエージェントの成果物は Fable 側で受け取り、レビュー・統合する。
5. サブエージェントへの指示は冗長な背景説明を避け、要点（対象・前提・受け入れ条件）に絞る。

## プロジェクト概要

**AtelierLoop** — ブラウザ完結（フロントエンドのみ）のイラスト練習ツール。出題 → 描く → 測る → 振り返るのループを1画面で回す。

- 詳細仕様は **`docs/SPEC.md`** が正。実装前に必ず該当フェーズのセクションを読むこと
- 完全フロントエンド（サーバー・APIキー不要、静的ホスティングで配信可能）
- LLM はコーチング専任、数値判断はアルゴリズム専任
- ストロークはベクタデータが正、ラスタは派生物

### 技術スタック

Vite + React + TypeScript / Canvas 2D（スタンプ方式ブラシ）/ Pointer Events API / One Euro Filter / Three.js / Transformers.js + WebGPU（Gemma 4 E4B）/ IndexedDB（idb）

### ディレクトリ構成

`docs/SPEC.md` §3 のとおり。`src/drawing/` はフレームワーク非依存（React を import しない）に保つこと。

### 開発コマンド

- `npm run dev` — 開発サーバー起動
- `npm run build` — 型チェック + プロダクションビルド
- `npm run preview` — ビルド成果物のプレビュー
